# free-worker — 個人事業・フリーランス業務支援デスク

デジタルコンテンツ主体の個人事業主・フリーランス向けの **ローカルファースト業務支援アプリ**。
EC・デジタル販売・サブスク・受託の事務(法令チェック / 商品・サブスク / 売上・請求 / 経費 / 契約・規約 / コンテンツ制作 / 顧客・同意 / バックアップ / 監査)を 1 つのローカルアプリで管理します。

## 設計の核 — 「クラウドAIが使えなくなっても全部動く」

- **ローカルファースト**: データは端末内の SQLite。サーバは `127.0.0.1` のみで待受け、外部に公開しません。
- **クラウドAI非依存**: AIなしで全機能が動作。AI補助は任意機能(既定OFF)で、許可するのは **ローカルLLM(Ollama / LM Studio)** と **Gemini Flash 無料枠** のみ。未接続でも一切困りません。
- **機密を外に出さない**: 実顧客データ・税番号・契約原本をクラウドへ送りません。
- オフライン動作は自動テスト(`packages/server/test/offline.test.ts`)が保証 — `fetch` を禁止した状態で全主要操作の成功を検証しています。

## 構成(npm workspaces モノレポ)

| パッケージ | 役割 |
|---|---|
| `@free-worker/core` | ドメインロジック・SQLite・マイグレーション・CSV・暗号化バックアップ・金額/日付ユーティリティ・監査・AIアダプタIF |
| `@free-worker/server` | Fastify(`127.0.0.1` 固定)・REST API・ビルド済み SPA の静的配信 |
| `@free-worker/web` | Vite + React + TS の SPA(全11画面 + ダッシュボード) |

## セットアップ・起動

```bash
npm install            # 依存導入(better-sqlite3 のネイティブビルドを含む)
npm run build:app      # core → web → server をビルド
npm run start          # http://127.0.0.1:4319 で起動(ブラウザで開く)
```

開発時(ホットリロード):

```bash
npm run dev            # サーバ(127.0.0.1:4319)
npm run dev:web        # 別ターミナルで Vite(127.0.0.1:5173, /api をプロキシ)
```

データ/ポートは環境変数で変更可能: `FREE_WORKER_DATA_DIR`(既定 `./data`), `FREE_WORKER_PORT`(既定 `4319`)。

## 開発・検証

```bash
npm run typecheck      # 全 workspace 型チェック
npm test               # vitest(ネットワーク不要)
npm run build          # 全ビルド
```

## 実装済み機能(EPIC)

- **EPIC-01** 事業プロフィール / **EPIC-02** 法令・税務チェックリスト(出典 S1〜S15 紐付け・`action_checklist.csv` シード)
- **EPIC-03** 商品・サブスク管理(**掲載前チェック**で特商法/動作環境/ライセンス/返品/継続課金条件の欠落を出典付き警告)
- **EPIC-05** 売上・請求(CSV入出力・月次集計) / **EPIC-06** 経費・証憑
- **EPIC-04** 契約・規約 版管理(publish 遷移) / **EPIC-07** コンテンツ制作WF(公開前チェック)
- **EPIC-08** 顧客・同意(email平文非保存) / **EPIC-12** 監査ログ(CSV出力)
- **EPIC-09** ダッシュボード(月次KPI・今月の期限)
- **EPIC-10** 暗号化ローカルバックアップ(aes-256-gcm + scrypt、復元テストログ)
- **EPIC-11** AI非依存モード(オフライン動作保証)

## 注意(重要)

法務・税務の最終判断は、税務署・税理士・弁護士・各公式窓口で確認してください。本アプリの文面・項目は実務メモであり、住所公開・インボイス登録・前払式支払手段・著作権/二次創作・成人向け/投資助言等に該当する商品は追加確認が必要です。出典は `docs/reference/sources.md` および `docs/compliance_sources.md` を参照。
